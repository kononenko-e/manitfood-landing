# Манит-бот — Конструктор кадра

Документ описывает онтологию «конструктора кадра» для Telegram-бота. Пользователь за 5 шагов собирает желаемый кадр, загружает фото своего блюда, бот через image-to-image генерирует результат.

---

## Главный принцип

**База референсов = одновременно и галерея для пользователя, и источник image-to-image промптов.**

Каждый кадр в базе:
- показывается пользователю как превью в карусели Telegram,
- используется как `Image 1` (референс стиля/ракурса/света/локации) при финальной генерации с `Image 2` (фото блюда пользователя).

В галерее бота — **только перегенерированные кадры** (уникальные, без копирайта). Исходники с Pinterest хранятся во внутренней папке `raw/`, пользователю не показываются.

---

## Ось 1: Жанр (ракурс / план камеры)

8 вариантов. Ракурс и композиция.

1. **Flat lay** — строго 90° сверху, всё разложено в плоскости
2. **45°** — классический «ресторанный» ракурс, три четверти
3. **Eye-level** — камера вровень с блюдом, кинематографично
4. **Macro** — экстремально близко, акцент на текстуре
5. **Hero** — один объект крупно, изолированно, минимализм
6. **Group** — несколько блюд / spread / фуршет
7. **Lifestyle / over-the-shoulder** — блюдо на переднем плане, рука/чашка/окно сзади
8. **Hands in action** — руки в кадре, процесс (готовка / наливание / разламывание)

---

## Ось 2: Стиль (настроение / эстетика)

12 вариантов. Свет + палитра + грейд + настроение.

1. **Тёмная драма** — 45° или close-up, single hard key light, chiaroscuro, глубокие тени, low-key, приглушённые тёмные тона
2. **Светлый и воздушный** — 45° или overhead, soft overcast daylight, ровная диффузия, high-key, светлый, минимум насыщенности
3. **Динамика / сплэш** — close-up / medium, hard directional + backlight, frozen action (брызги, льющийся соус, падающие ингредиенты), диагональная энергия, vibrant
4. **Флэтлей редакторский** — overhead flat-lay, ровный мягкий верхний свет, графичная сетка, контролируемая ограниченная палитра
5. **Домашний рустик** — eye-level / 45°, тёплый оконный свет, естественная небрежность, тёплые земляные тона
6. **Fine dining редакторский** — close-up / 45°, единичный spotlight, минимализм, центр, negative space, тёмная сдержанная палитра
7. **Винтажная плёнка** — eye-level / 45°, тёплый tungsten / golden hour, Kodak Portra / Fuji Velvia эмуляция, зерно, выцветшие тёплые тона
8. **Монохром / дуотон** — любой план, часто close-up, жёсткий направленный свет, ч/б или ограничение до 2 hex-цветов
9. **Пастельная нежность** — eye-level / 45°, close-up для десертов, мягкий рассеянный, центр, пастель (розовый / кремовый)
10. **Яркий поп-арт** — overhead / eye-level, яркий ровный почти frontal ring light, графичная смелая композиция, высокая насыщенность
11. **Макро-текстура** — extreme close-up, мягкий направленный, плотный кроп на текстуру, акцент на specular highlights
12. **С руками / лайфстайл** — 45° / eye-level, medium, мягкий естественный свет, руки в кадре (наливают, держат, отламывают), тёплая натуральная палитра

**Свет и палитра зашиты в стиль** — отдельными осями не выносятся. Если нужно изменить — на этапе правок.

---

## Ось 3: Локация / сцена

Три типа, все нужны. Пользователь выбирает тип → внутри типа конкретный вариант.

### Тип A: Готовая сцена
Блюдо внутри полноценной 3D-среды, видны детали интерьера/окружения.

- Домашняя кухня
- Ресторанная подача
- Кофейня / бар
- Outdoor: терраса
- Outdoor: пикник
- Outdoor: пляж
- Праздничный стол
- Рынок / food market

### Тип B: Поверхность
Плоский фон — блюдо лежит на чём-то, окружение минимально.

- Светлое дерево (farmhouse)
- Тёмное дерево / мореный дуб
- Белый мрамор с прожилками
- Чёрный мрамор / сланец
- Бетон / серый камень
- Белая столешница / нейтральная поверхность
- Крафт-бумага / пергамент
- Льняная скатерть
- Клетчатая ткань
- Текстура крупным планом (дерево / камень)

### Тип C: Студийный фон
Бесшовный контролируемый фон, без окружения.

- Однотонный светлый
- Однотонный тёмный
- Цветной (пастель)
- Цветной (яркий)
- Градиент

---

## Ось 4: Реквизит / props

6 уровней — что лежит/стоит рядом с блюдом.

1. **Минимум** — только блюдо на тарелке / доске, ничего лишнего
2. **Утварь** — вилка / нож / ложка / салфетка, чисто сервировка
3. **Ингредиенты-компаньоны** — помидоры рядом с пастой, ягоды с десертом, лимон с рыбой
4. **Напиток в паре** — бокал вина, чашка кофе, коктейль, но без человека
5. **Тематический декор** — книга, свеча, цветы, газета, специи в баночках
6. **«Грязная» / живая подача** — крошки, брызги соуса, разлитое вино, откусанный кусок

---

## Ось 5: Формат / соотношение сторон

5 вариантов.

- **1:1** — Instagram пост, Авито, маркетплейсы
- **4:5** — Instagram portrait, каталог
- **9:16** — Stories, Reels, TikTok
- **16:9** — YouTube, горизонтальный баннер
- **2:3** — Pinterest pin, обложка

---

## Дополнительные оси (НЕ в основном флоу, выносятся на этап правок)

- **Свет** — направление, жёсткость, температура (по умолчанию = как в стиле)
- **Палитра / грейд** — тёплая / холодная / монохром / высококонтрастная (по умолчанию = как в стиле)
- **Дополнительная активность** — сплэш / пар / разрезание / наливание

Эти ручки показываются **после первой генерации**, чтобы не перегружать пользователя на старте.

## UX-маршрут пользователя

### Шаг 0: Онбординг
Бот приветствует, объясняет формат за 5 шагов, **параллельно** просит загрузить фото блюда (загружается сразу, чтобы не блокировать конструктор).

### Шаг 1: Жанр
Карусель из 8 превью (Telegram поддерживает карусель фото с кнопками под ними после обновления).

Каждое превью = миниатюра реального кадра из базы, размеченного по жанру.

Под каруселью кнопка: **«Не знаю — подберите за меня»**.

### Шаг 2: Стиль
Карусель из 4-6 превью, **отфильтрованная** по выбранному жанру. Под каруселью снова кнопка автовыбора.

### Шаг 3: Локация
Сначала выбор типа (3 таба): **Готовая сцена / Поверхность / Студия**.

Внутри таба — карусель конкретных локаций, **отфильтрованная** по жанру + стилю.

### Шаг 4: Реквизит
Карусель из 6 превью уровней реквизита.

### Шаг 5: Формат
Кнопки в ряд: **1:1 / 4:5 / 9:16 / 16:9 / 2:3**.

### Финал
Бот: «Собираю кадр… ⏳» → генерация → результат.

Под результатом кнопки:
- **Скачать**
- **Ещё вариант** (перегенерация с тем же референсом и блюдом)
- **Изменить** → вызывает ручки правок (локация / реквизит / свет / палитра / активность)

## Итоговый image-to-image промпт

Когда пользователь выбрал референс и загрузил фото блюда, бот формирует запрос к модели:

```
Image 1 = выбранный референс (стиль/ракурс/свет/локация/реквизит)
Image 2 = фото блюда пользователя

Промпт: "Use the composition, lighting, angle, mood, color grading
and props from Image 1. Replace the dish with the dish from Image 2.
Keep all styling, background, tableware, palette, atmosphere from
Image 1. Final image in {format} aspect ratio."
```

Без явного разделения «кто есть кто» модели иногда смешивают оба изображения в кашу — поэтому фраза «Use ... from Image 1, replace with ... from Image 2» обязательна.

## Структура базы референсов

Каждый кадр в базе — размеченный multi-label объект:

```json
{
  "id": "ref_001",
  "preview": "references/ref_001_thumb.jpg",
  "reference": "references/ref_001.jpg",
  "source_pinterest": "https://...",
  "genre": "flat_lay",
  "style": "bright_airy",
  "location_type": "surface",
  "location": "white_marble",
  "props": "minimal",
  "format": "1:1",
  "tags": ["breakfast", "smoothie_bowl"]
}
```

### Объём MVP

- 8 жанров × ~30 кадров = 240
- 12 стилей × ~30 кадров = 360 (с учётом пересечений ~150-200 уникальных)
- ~30 локаций × ~20 кадров = 600 (с пересечениями ~200-300 уникальных)
- 6 уровней реквизита × ~20 кадров = 120

**Реально уникальных кадров с учётом пересечений по всем осям: ~400-500** — комфортный размер для MVP.

### Физическая раскладка

```
genre/         ← 8 папок
style/         ← 12 папок
location/scene/     ← подпапки локаций
location/surface/   ← подпапки поверхностей
location/studio/    ← подпапки студийных фонов
```

Один и тот же файл кадра может индексироваться сразу в нескольких категориях (multi-label через `db/references.json` + дубли symlink в нужных папках, либо только через JSON без дублей).

## Умные фичи (закладываем в архитектуру, реализуем постепенно)

1. **«Подберите за меня»** — кнопка на каждом шаге конструктора, бот сам выбирает на основе трендов / популярного
2. **«Случайный кадр»** — для тех, кто хочет удивиться
3. **Запоминание предпочтений** — если пользователь в прошлый раз выбирал «Тёмная драма», предлагать её первой
4. **Сохранённые комбинации** — пользователь собрал удачный кадр, сохранил как шаблон, потом применяет к другим блюдам одной кнопкой
5. **A/B по жанрам/стилям** — бот показывает, какие кадры чаще выбирают (помогает наполнять базу тем, что реально нужно)

---

## Что НЕ входит в этот документ (осознанно)

- Матрица совместимости осей (жанр×стиль, жанр×локация, стиль×локация, стиль×реквизит) — будет реализована в коде фильтрации каруселей
- Конкретный список «какие ручки выносим на этап правок» — продумаем отдельно
- Поведение бота при пустой комбинации (нет подходящих кадров) — продумаем отдельно
- Парсер Pinterest, конвейер перегенерации, структура БД — это уже код, следующий этап
- Конкретный объём локаций в типе «Готовая сцена» (сейчас 8, может расширяться) — финализируем отдельно
