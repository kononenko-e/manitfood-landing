# Манит-бот — Матрица совместимости осей

Бинарная логика: **✓ = совместимо** (показываем в карусели следующего шага), **✗ = несовместимо** (прячем). Правило «clash-only» — исключены только реально конфликтующие сочетания (по ракурсу / свету / характеру сцены), остальное разрешено.

Каскад выбора: **жанр → стиль** (фильтр по жанру) → **локация** (фильтр по жанр + стиль) → **реквизит** (фильтр по стилю) → **формат** (свободно).

Машиночитаемая версия — `compatibility.json`, логика фильтрации — `compatibility.js`.

Легенда стилей: DD — тёмная драма · BA — светлый/воздушный · DS — динамика/сплэш · EF — флэтлей редакторский · HR — домашний рустик · FD — fine dining · VF — винтажная плёнка · MO — монохром · PA — пастель · PO — поп-арт · MT — макро-текстура · HL — с руками/лайфстайл.

## Как это ложится в код / БД

Каждый кадр в базе размечен по всем осям (`genre`, `style`, `location_type`, `location`, `props`, `format`). Матрицы выше нужны **до** генерации базы — чтобы не парсить/не хранить заведомо несовместимые сочетания, — и **во время** флоу, чтобы карусель шага N+1 показывала только разрешённые значения.

Два способа применять матрицу к карусели:

1. **По матрице (предиктивно):** список кнопок шага берётся прямо из `compatibility.json` (`stylesForGenre`, `locationTypesFor`, `propsForStyle`). Быстро, не зависит от наполнения базы, но может предложить значение, под которое ещё нет ни одного кадра.
2. **По базе (фактически):** `SELECT DISTINCT style FROM refs WHERE genre = ?` — показываем только то, под что реально есть кадры. Всегда непустая карусель, но требует наполненной базы.

Рекомендация: **пересечение обоих** — матрица задаёт, что *логически можно*, база отсекает, чего *ещё нет*. Тогда пользователь не упирается в пустую карусель (см. открытый вопрос «поведение при пустой комбинации» в исходном документе).

Пустых тупиков в самой матрице нет: для любого жанра ≥ 5 стилей, для любой пары жанр+стиль ≥ 1 тип локации, для любого стиля ≥ 2 уровня реквизита (проверено прогоном всех каскадов).
